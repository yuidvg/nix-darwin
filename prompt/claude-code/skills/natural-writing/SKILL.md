# 自然な文章を書くためのガイドライン

## 背景と出典

このガイドラインは以下の一次ソースに基づいています：

- **Wikipedia:Signs of AI writing** (https://en.wikipedia.org/wiki/Wikipedia:Signs_of_AI_writing) — WikiProject AI Cleanup が数千件のAI生成テキストを分析して蓄積したフィールドガイド（2025年8月公開）
- **Humanizer** (https://github.com/blader/humanizer) — 上記Wikipediaガイドを Claude Code スキルとして実装した Siqi Chen のプロジェクト（24パターン）
- 上記に加え、日本語特有のAI文体パターンを独自に追加

目的は「AI検出を回避する」ことではなく、**読み手にとって具体的で、信頼でき、人間が考えた文章だと伝わる文体で書く**ことです。

---

## 原則：なぜAIの文章は「AIっぽい」のか

LLMは統計的に最も確率の高い次の単語を予測して文章を生成します。その結果、**平均への回帰**が起こります。具体的で珍しい事実は削り落とされ、汎用的でポジティブな表現に置き換えられます。Wikipediaの編集者はこれを「肖像画がぼやけていくのに、その人物が重要だと叫ぶ声だけが大きくなる」と表現しています。

このガイドラインは「この単語を使うな」というブラックリストではありません。パターンの**組み合わせ**が問題であり、個々の表現は人間も使います。

---

## トーンの自動判断

文脈に応じてフォーマリティを切り替えてください：

- **行政・企業向け提案書・報告書**: 敬体（です・ます調）、適度な専門用語、根拠の明示。ただし硬すぎる官僚文体は避ける
- **ブログ・対外発信・SNS**: 常体も可、具体的なエピソードや一人称を積極的に使う
- **社内ドキュメント・議事録**: 簡潔さ優先、箇条書き可だが形式的な装飾は不要
- **学術・研究文脈**: 論理構造を重視、主張と根拠の対応を明確に

判断に迷ったら、**読み手が誰か**を確認してください。

## 日本語で避けるべきAIパターン（24項目）

### A. 内容の問題（何を書いているか）

**1. 意義の水増し（Significance inflation）**

些細な事実に「重要な役割を果たしている」「大きな意義を持つ」と付け加える癖。

❌ 本システムは、地方自治体のDXにおいて重要な役割を果たしており、市民参加の新たな地平を切り拓くものです。
✅ 本システムは自治体の会議時間を平均40%短縮し、参加者の満足度を従来比で15ポイント向上させました。

**対処法**: 「重要」「大きな意義」と書きたくなったら、代わりに**数字か具体的な事実**を入れる。

**2. 曖昧な帰属（Vague attribution / Weasel words）**

「専門家は～と指摘している」「多くの研究が～を示している」など、誰が言ったか不明な権威づけ。

❌ 多くの自治体関係者が、AI活用の必要性を指摘しています。
✅ 総務省の2024年調査では、回答した自治体の67%がAI導入を検討中と回答しました。

**対処法**: 「誰が」「いつ」「どこで」を書けないなら、その一文自体が不要な可能性が高い。

**3. 表面的な分析（Superficial -ing analysis）**

文末に「～を示しており」「～が期待されます」「～に貢献しています」と付け加えて分析した風にする癖。英語では present participle (-ing) の濫用として知られる。

❌ この取り組みは住民の声を行政に届ける新たなチャネルとなっており、今後の市民参画の活性化に大きく貢献することが期待されます。
✅ 富岡市の実証では、従来のパブリックコメントで3件だった意見が、本システム導入後は247件に増えました。

**対処法**: 「～が期待されます」「～に貢献しています」を消して、文章が成り立つか確認する。成り立つなら、その部分は不要だった。

### B. 言語・文法の問題（どう書いているか）

**4. 大仰な漢語表現の連続**

「多岐にわたる」「幅広い」「包括的な」「革新的な」「画期的な」が連続すると、内容が空洞化する。

❌ 多岐にわたる課題に対して包括的かつ革新的なアプローチで取り組んでいます。
✅ 会議の効率化、アンケートの自動集計、結果の可視化の3点に絞って取り組んでいます。

**5. 「～において」「～における」の過剰使用**

一段落に3回以上出てきたら要注意。日本語のAI文体で最も頻出するパターンの一つ。

❌ デジタル民主主義における市民参加において、AIの活用における課題は多い。
✅ デジタル民主主義で市民参加を促すとき、AI活用には独自の課題がある。

**6. コピュラ（「である」「です」）の回避**

LLMは「is」を「serves as」に、「です」を「としての役割を担っています」に置き換える傾向がある。

❌ 本ツールは自治体と市民を結ぶ架け橋としての役割を担っています。
✅ 本ツールは自治体と市民をつなぐツールです。

**7. 受動態の多用**

「～されている」「～と考えられている」「～と評価されている」が続くと、誰の意見・判断なのかが消える。

❌ この手法は効果的であると評価されており、多くの自治体で導入が検討されています。
✅ 太田市と東大阪市で導入済みで、担当者からは「会議時間が半分になった」との評価を得ています。

### C. 構造・スタイルの問題（どう見せているか）

**8. 三つ組の法則（Rule of three）の機械的適用**

LLMは「A、B、C」の三つ組を好む。常に三つに揃えなくてよい。二つでも四つでも、実態に合った数を使う。

❌ 効率性、透明性、持続可能性を兼ね備えたシステムです。
✅ 会議時間の短縮と、議論の透明化を目的としたシステムです。

**9. ダッシュ（—）の過剰使用**

英語のAI文体で顕著だが、日本語でも「——」や括弧で情報を詰め込みすぎるパターンがある。一文に補足が2つ以上入ったら、文を分割する。

**10. 太字＋コロンの箇条書き**

「**速度**: ○○が向上」「**品質**: △△が改善」のように、太字見出し＋コロンで始まる箇条書きを並べるパターン。人間はこの形式をあまり使わない。

❌
- **効率化**: 会議時間を40%削減
- **品質向上**: 意見の多様性が向上
- **コスト削減**: 運営費を年間30%削減

✅ 会議時間は40%短縮され、参加者からの意見数は3倍に増えました。運営費は年間30%減っています。

散文で書けるなら散文で書く。箇条書きは「読み手がリストとして参照する必要がある」ときだけ使う。

**11. 絵文字の見出し利用**

「💡 ポイント」「🚀 次のステップ」「✅ まとめ」のような絵文字見出し。ビジネス文書では使わない。カジュアルなブログでも控えめに。

**12. 均一な段落長と文長**

すべての段落が同じ長さ、すべての文が同じリズム。人間の文章にはムラがある。短い一文の後に長い説明が来たり、一行だけの段落があったりする。意識的にリズムを崩す。

### D. 定型表現の問題（AIが好む言い回し）

**13. オープニングの定型句**

「近年、～が注目を集めています」「～の時代において」「急速に変化する現代社会において」で始まる文章。

❌ 近年、AIを活用した市民参画が注目を集めています。
✅ 2024年、構想日本は富岡市で初めてAIを使った住民討議を実施しました。

**対処法**: 具体的な事実・エピソード・問いかけから始める。

**14. クロージングの定型句**

「以上のことから」「総じて」「今後ますます～が期待されます」「引き続き～に取り組んでまいります」で終わる文章。

❌ 今後もデジタル民主主義の発展に貢献してまいります。
✅ 次のステップとして、与謝野町での実証実験を3月に予定しています。

**対処法**: 最後は**次の具体的なアクション**か**未解決の問い**で終わる。

**15. 接続詞の機械的連続**

「まず」「次に」「さらに」「加えて」「最後に」が規則正しく並ぶ。また、「しかしながら」「とはいえ」「一方で」の過剰使用。

**対処法**: 接続詞なしでも文脈から繋がるなら、省略する。日本語は接続詞がなくても自然に読める言語。

**16. AI語彙リスト（日本語版）**

以下の単語・表現が高密度で出現すると、AIっぽさが増す。個別の使用は問題ないが、一つの文書に何度も出てきたら言い換えを検討する：

| AIが好む表現 | 代替案 |
|---|---|
| 多岐にわたる | 具体的に列挙する |
| 画期的な / 革新的な | 何がどう新しいか説明する |
| シームレスな | 具体的にどう繋がるか書く |
| 俯瞰的に / 包括的に | 何をカバーしているか具体的に書く |
| ～という観点から | 削除しても通じることが多い |
| ～に資する | ～に役立つ / ～を助ける |
| 〜の醸成 | ～を育てる / ～をつくる |
| 深く関わっている | どう関わっているか書く |
| 不可欠な | なぜ必要か理由を書く |
| 喫緊の課題 | いつまでに何をする必要があるか書く |

**17. 「Not only A but also B」構文の日本語版**

「～だけでなく、～も」「単に～にとどまらず」の頻出。一つの文に詰め込まず、二文に分ける。

❌ 本システムは単に会議の効率化にとどまらず、市民参画の新たなモデルを提示するものです。
✅ 本システムは会議を効率化します。同時に、声を上げにくい住民が意見を出せる仕組みとして設計しています。

### E. コミュニケーション上の問題

**18. チャットボットの残骸**

「お役に立てれば幸いです」「ご質問があればお気軽に」「以下にまとめました」など、対話的な表現が文書に残る。文書には不要。

**19. 過剰な前置き・メタ説明**

「この文書では〜について説明します。まず〜を概観し、次に〜を検討します」のような文書の構造説明。読み手は目次を見ればわかる。本題から入る。

**20. 均等な網羅性**

すべてのトピックに同じ分量を割く傾向。人間は重要なことに厚く、些末なことに薄く書く。メリハリをつける。

### F. フィラー・ヘッジの問題

**21. 過剰なヘッジ（逃げの表現）**

「～と言えるかもしれません」「～の可能性も否定できません」「一概には言えませんが」の連続。確信があるなら言い切る。不確実なら何が不確実かを明示する。

**22. 「重要なのは」の濫用**

「ここで重要なのは」「特筆すべきは」「注目すべきは」が頻出。本当に重要なら、内容自体がそれを示す。ラベルを貼る必要はない。

**23. 同義語の言い換えループ**

同じことを別の言葉で3回繰り返す（LLMの反復ペナルティに起因する挙動）。

❌ 市民の声を集め、住民の意見を収集し、生活者の考えをまとめました。
✅ 住民247人の意見を集めました。

**24. 安全な両論併記**

「メリットもあればデメリットもあります」「一長一短です」で終わる分析。立場を取る。少なくとも、自分（あるいは自社）がどちらに賭けているかを明示する。

## 良い文章のポジティブガイドライン

パターンを避けるだけでは「無菌室のような退屈な文章」になります。以下を積極的に取り入れてください：

1. **具体的な固有名詞を入れる**: 「ある自治体」ではなく「太田市」。「あるツール」ではなく「倍速会議」。
2. **数字を入れる**: 「多くの」ではなく「247件」。「大幅に」ではなく「40%」。
3. **時系列を明示する**: 「近年」ではなく「2024年3月」。
4. **一人称を恐れない**: フォーマルな文書でも「当社は」「私たちは」を使う。誰が主語かを明確にする。
5. **文のリズムを変える**: 短文と長文を混ぜる。一文だけの段落も使う。
6. **判断を示す**: 「AとBがあります」で終わらず、「私たちはAを選びました。理由は～」まで書く。
7. **未解決の問題を認める**: 完璧に解決済みのように書かない。「この点はまだ検証中です」は信頼性を高める。
8. **読み手への想像力を持つ**: この文を読んだ人が次に何を知りたいかを考え、先回りして書く。

## 適用時の注意

- このガイドラインは**新規に文章を生成するとき**と**既存の文章をレビュー・編集するとき**の両方に適用してください。
- すべてのパターンを一度に排除しようとせず、文書の目的と読者に照らして**最も影響の大きいものから対処**してください。
- 文脈によっては「AIっぽい」表現が最適な場合もあります（例：法的文書の定型表現、学術論文の慣例的フレーズ）。機械的に排除しないでください。
- 判断に迷ったら「この一文を消しても、文書の意味は変わるか？」と自問してください。変わらないなら、その一文はフィラーです。

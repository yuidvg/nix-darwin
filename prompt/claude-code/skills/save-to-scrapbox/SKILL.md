# Save to Scrapbox

会話中のコンテンツをScrapboxに書き込むスキル。

トリガー: "Scrapboxに保存して", "スクボに入れて", "メモしておいて", "保存しておいて", "save to scrapbox", "ブックマークして", "scrapboxに記載して"

また、googleドライブは、社外に共有するものだけ入れて、基本的に更新されるドキュメントはscrapboxに置いておく。

## 対応プロジェクト

| プロジェクト名 | 用途 | 使い分け |
|---|---|---|
| `plural-reality` | 多元現実のチームナレッジベース（デフォルト） | 特に指定がなければこちら |
| `tkgshn-private` | 高木の個人メモ | 「個人の」「プライベートに」と言われた場合 |

## コマンド

`scrapbox-write` は stdin からページ本文を受け取り、WebSocket 経由で Scrapbox に書き込む。

| モード | フラグ | 動作 |
|---|---|---|
| 全置換 | （デフォルト） | ページ本文を stdin で丸ごと置き換える |
| 末尾追記 | `--append` / `-a` | 既存ページの末尾に stdin を足す。未作成なら新規作成 |
| 先頭追記 | `--prepend` / `-P` | タイトル直下（ページ上部）に stdin を挿入する。逆時系列の追記用 |
| プレビュー | `--dry-run` / `-n` | 書き込まず、実際に送る行を stdout に出す（`SCRAPBOX_SID` 不要） |

```bash
cat <<'BODY' | scrapbox-write -p plural-reality -t "ミーティングメモ"
議題
 項目1
 項目2
BODY
```

- 長い本文・既存ページへの追記は、まず `--dry-run` で出力を確認してから書き込む。
- 成功時に stdout に出る Scrapbox URL を、必ずユーザーに報告する。
- `scrapbox-write` は各行に半角スペース1つを足す（新規ページ向け）。**既存ページの行を in-place で書き換える**ときは、行頭のインデント（tab/space）が崩れるため scrapbox-write を使わず、`@cosense/std` の `patch` で現在行を取得→変換→返す方式にする（後述「既存ページのLLMマーク」）。

## 書き方ルール（基本）

Scrapbox記法で書く（Markdownではない）。読みやすさは「ネストした箇条書きで構造化されているか」で決まる。

### 1. 段落で書かず、箇条書きで構造化する
- 長い文を地の文（パラグラフ）で書かない。要素ごとに改行し、ネストで階層化する。
- 1行1要素。「主語 → 結論」「事実 / 補足」のように短く切る。箇条書きは頻繁に使う。

❌ パラグラフ + 文中強調:
```
[✅契約] に基づく支払いについて、青山が米国在住である点を踏まえると [** 非居住者] に該当し、[** 源泉徴収義務] が発生する可能性がある。
```
✅ 箇条書きに分解（強調なし）:
```
背景
 [✅契約] に基づく支払い
 青山は米国学生ビザで在住 → 日本の税法上は非居住者に該当
 非居住者への著作物使用料 → 源泉徴収義務が発生する可能性
```

### 2. インデントの仕組み・空行
- `scrapbox-write` は各行に半角スペース1つを足す。行頭スペースなし → インデント1段。ネストは行頭スペースを足す。
- **セクション間の空行は使ってよい**（中黒は出ない）。読みやすさのため適度に空ける。

### 3. 強調（太字）を文中で使わない
- 文の途中の語を `[* …]` `[** …]` `[[ …]]` で強調しない。不自然になる。
- 構造はネストで表現する。見出しが要るときだけ行頭ラベルにする。

### 4. テーブル内では強調を使わない
- テーブルのセルでは強調記法が効かない。プレーンテキストで書く。
- `table:名前` の次行から、1段深いインデント + TAB区切りで行を書く。

### 5. `#` を使わない
- `#3` のような番号・記号に `#` を使わない。Scrapbox ではタグ/リンク化されてしまう。
- 番号は `No.3` `3番目` のように書く。

### 6. ステータス記号（TODO）
チェックや進捗には絵文字を使う（`[ ]` `[x]` はリンク記法と衝突するので使わない）。
- `⬜` タスク / `⏳` 進行中 / `⏹️` 停止 / `☑️` 完了

### 7. 関連ページをリンクする
- 似た語彙・文脈のページは既に存在するはず。`[ページ名]` でリンクして知識グラフに繋ぐ。
- 書く前に Scrapbox を検索して実在ページを確認する（`/api/pages/<project>/search/query?q=…`、cookie に `SCRAPBOX_SID`）。出典元は冒頭に `from [元ページ]`。

## LLMが書いた文章のマーク（最重要）

人間が書いた文章と LLM が書いた文章を視覚的に区別する。LLM（あなた）が生成した文は **`[( …]` 装飾**で囲む。

- `[( …]` は UserCSS `.deco-(`（薄く表示する装飾）。**リンクではない**ので幽霊ページは作られず、**中に `[リンク]` も入れられる**（例 `[( [冷蔵庫]: 27,000円]`）。
- **アイコンの付け方**:
  - 1行だけの LLM 記述 → `[( …]` のみ（装飾で LLM とわかるので**アイコン不要**）。
  - 複数行にわたる LLM ブロック → **先頭行にだけ agent アイコン**を付け、続く行も**全部 `[( …]`** で書く。
- **agent アイコンは書いた主体で使い分ける**:
  - Claude Code が書く → `[claude code.icon]`
  - Codex が書く → `[codex.icon]`
- **タスクは page object 化する**: AI が生成するタスクは `[( [⬜ タスク名]]` のように内側を `[…]` リンクにする。そうしないと他ページから参照できる object にならない。
  - 例: `[( [⬜ 椅子を中古で探す]]` / `[( [⏳ 見積もり依頼中]]`

### provenance（実質的な生成は出自を残す）
日付を最上位レイヤーに置き、「誰が・どんなプロンプトで書かせたか」を引用ブロックで明記してから、LLM 生成内容を書く。
```
[2026/05/29]
 >[tkgshn.icon]による指示:
 > 「<ユーザーが出した実際のプロンプト>」
 [( ここからの概要は Claude Code が生成][claude code.icon]
  [( 要点を箇条書きで…]
  [( [関連ページ] への参照もリンクのまま装飾できる]
```
- 日付ブラケット `[YYYY/MM/DD]` が最上位、その下に概要を箇条書きで展開する（日付ベースの動的Wiki）。
- 指示の引用 `>` 行は人間の言葉なので装飾しない。LLM が生成した行だけ `[( …]`。
- プロンプト引用に `[( ` や `[…]` をそのまま入れると壊れるので、その2文字は引用内で言い換える。

### 既存ページのLLMマーク（人間の記述への補足）
人間が書いたページに LLM が補足した箇所だけをマークする。
- 人間/LLM の区別は ground truth で判断する（同一 Scrapbox アカウントで書くと行の `userId` では判別できない。セッションログや実際の patch 内容で確定する）。
- 人間の項目への単行補足は、項目はそのまま残し、**1段下げて `[( …]`**（単行なのでアイコンなし）:
```
[ブレインスリープのAll in One]
 [( 新品: 77,000円 / 中古: 3.0〜4.5万円目安]
```
- 複数行の LLM ブロックは先頭にアイコン、全行 `[( …]`。
- in-place 編集なので `scrapbox-write` ではなく `@cosense/std` の `patch` で現在行を変換する（インデント保持のため）。

### 人間が編集したら人間の色に戻す
- `[( …]` の装飾を外せば普通の人間テキスト色になる。手動なら `[( X]`→`X`、`[( [X]]`→`[X]`（リンク/タスクは保持）。
- UserScript の「人間化」ボタンで選択行を一括変換できる（`/tkgshn-extension` に設置）。CSS 単体では編集者を判定できないため、外すのは UserScript か手動。

## 記法リファレンス
- 見出し（行ラベル）: `[* 見出し]`（大きく: `[*** 見出し]`）
- リンク: `[ページ名]` / 外部リンク: `[URL 表示名]`
- アイコン: `[tkgshn.icon]` / `[claude code.icon]` / `[codex.icon]`
- LLMマーク装飾: `[( テキスト]`（`.deco-(` = 薄表示。リンク内包可）
- 箇条書き: 行頭スペースでインデント
- 引用: 行頭 `>`
- コード: インライン `` `code` `` / ブロック `code:filename` + インデント
- テーブル: `table:名前` + 1段深いインデントで TAB区切り行
- 打ち消し線: `[- 取り消すテキスト]`

## URL・参考リンクの書き方
- 単に URL を貼るだけなら、メタデータが無くても URL のままでよい（Scrapbox 側でカード表示される）。
- 「参考リンク」として載せるときは、WebFetch でページタイトル（メタデータ）を取得し `[URL タイトル]` 形式にする。取得できなければ URL のまま。

## 追記・改訂のルール

### 部分的な差し替え（参考リンク・一文の更新）
古い記述を打ち消し線で残し、1段下げて日付付きで追記する。
```
参考リンク
 [- 旧ソース: ○○のガイド — https://old.example.com]
  [2026/5/29] 追記: [https://new.example.com 新ソース: ××の解説] に差し替え
```
- 打ち消し線の中にライブなリンクを入れ子にしない。旧リンクは「タイトル — URL」の素のテキストで打ち消す。

### ページ全体の内容が変わったとき
1. 逆時系列: 新しい情報・追記はページの上の方に足していく（`--prepend`）。
2. 関連ページへの波及: 概要が変わったら、`[…]` でリンクしている関連ページに残る古い情報も打ち消し線で消し、その下に `[このページ名] に追記あり` と書いて誘導する。

## その他の注意
- タイトル行は本文に含めない（`--title` で渡したものが自動で1行目になる）。絵文字付きタイトルも `--title` 側に書く。
- 追記と全置換を混同しない。末尾に足すなら `--append`、上部に足すなら `--prepend`、丸ごと差し替えるなら デフォルト（replace）。

## ツイートURLの場合

ツイートURLを保存する場合は、twitter-bookmark API 経由:

```bash
curl -s -X POST http://localhost:3000/api/bookmark \
  -H "Content-Type: application/json" \
  -H "x-api-key: $TWITTER_BOOKMARK_API_KEY" \
  -d '{"url": "ツイートのURL"}'
```

前提: `twitter-bookmark` サーバーが起動中であること。

# Ask Network — 人脈を活かした情報収集・紹介メッセージ作成

Scrapbox のナレッジベースからユーザーの人脈を探索し、特定のコンテキスト（相談内容・依頼・紹介）に対して、それぞれの人物に合わせたパーソナライズドメッセージを作成する。

## トリガー

### 自動トリガー

- ユーザーが「誰かに聞きたい」「紹介してほしい」「この分野で知り合いいないかな」と言ったとき
- 外部からの相談メールやメッセージが共有され、「誰か紹介できないか」と聞かれたとき
- 「〜に詳しい人を探して」「〜について相談できる人いる？」

### 手動トリガー

- `/ask-network`
- 「ネットワークで探して」「人脈から探して」

## 実行手順

### Phase 1: コンテキストの把握

ユーザーから以下を確認する（会話の流れで自然に把握する。足りない情報だけ聞く）:

1. **何についての相談か**: メール原文、質問内容、探している専門性
2. **誰に送るか**: 具体的な人名が出ているならそれを使う。出ていなければ Phase 2 で探す
3. **トーン**: カジュアル？フォーマル？（デフォルト: 相手との関係性に応じて調整）
4. **出力先**: Facebook、Slack、メール、LINE など（デフォルト: 汎用テキスト）

### Phase 2: Scrapbox から候補者を探索

ユーザーが具体的な人名を挙げていない場合、コンテキストからキーワードを抽出し、Scrapbox で関連人物を探す。

#### 検索戦略

1. **トピックベースの検索**: コンテキストのキーワードで全文検索

```bash
/bin/bash -c 'curl -s "https://scrapbox.io/api/pages/tkgshn-private/search/query?q=KEYWORD&limit=10" \
  -H "Cookie: connect.sid=SCRAPBOX_SID"'
```

2. **人物ページの特定**: 検索結果から人物ページを抽出（名前っぽいタイトルを判別）

3. **両プロジェクトを検索**: `tkgshn-private` と `plural-reality` の両方で探す

#### 候補者の提示

見つかった人物を一覧にしてユーザーに提示し、誰にメッセージを送るか確認する:

```
以下の人物が関連しそうです:
1. Aさん — [所属] / [なぜ関連するか一言]
2. Bさん — [所属] / [なぜ関連するか一言]
3. Cさん — [所属] / [なぜ関連するか一言]

誰にメッセージを送りますか？追加したい人はいますか？
```

### Phase 3: 人物背景の深掘り（2hop検索）

送信先が決まったら、**各人物に対して並列で** Scrapbox の 2hop 検索を実行する。

```bash
# 各人物のページを 2hop で取得（並列実行）
/bin/bash -c 'curl -s "https://scrapbox.io/api/smart-context/export-2hop-links/tkgshn-private.txt?title=PERSON_PAGE_TITLE" \
  -H "Cookie: connect.sid=SCRAPBOX_SID"'
```

各人物について以下を把握する:

| 項目 | 具体例 |
|---|---|
| **現在の所属・役職** | 「〇〇大学 教授」「〇〇社 CTO」 |
| **ユーザーとの関係の深さ** | よく会う / たまに相談する / 知人程度 |
| **最後のやり取り** | 「2025/1/22 に MTG で同席」 |
| **相談テーマとの関連性** | なぜこの人に聞くと良いのか |
| **期待できる 2hop 先** | この人が知っていそうな人・組織 |

**重要**: サブエージェント（Agent ツール）を使って並列調査する。メインコンテキストを汚さない。

### Phase 4: メッセージ生成

各人物に対して、以下の原則でメッセージを作成する:

#### メッセージ設計の原則

1. **関係性に応じたトーン**:
   - 親しい相手 → カジュアル（「お疲れ様です」「ちょっと相談なんだけど」）
   - ビジネス上の知人 → 丁寧だが距離感あり（「ご無沙汰しています」「突然のご連絡で恐縮ですが」）
   - 目上・先生 → フォーマル（「お世話になっております」）

2. **自分の立場を明示**: 「僕自身は〇〇の文脈で関わってきたが、△△は専門外」

3. **相手にお願いする理由を具体的に**: 「〇〇の経験がある△△さんなら...」（Scrapbox で調べた背景を反映）

4. **期待値を低く設定**: 「直接でなくても、"この人なら知ってるかも"レベルで十分」

5. **2hop 先の紹介でもOKと伝える**: 「ご自身でなくても、心当たりのある方がいれば」

6. **コンテキストは簡潔に**: 相手が読んで30秒で状況を把握できる長さ

#### メッセージの構造

```
[挨拶 — 関係性に応じたトーン]

[依頼の背景 — 1-2文で状況説明]

[具体的に何を聞きたいか — 箇条書きまたは短文]

[なぜあなたに聞くのか — 相手のバックグラウンドに触れる]

[期待値の調整 — 「直接でなくても紹介先でも助かります」]
```

#### 汎用投稿文も作成

Facebook や Slack の広いコミュニティに投げる用の汎用文も1つ作成する:
- 自己紹介は不要（既に知っている人が読む前提）
- 何を探しているかを簡潔に
- 元の相談のコンテキストを適度に含める

### Phase 5: ファイル出力

**必ずファイルに書き出す**（ターミナル直接出力はコピペが面倒なため）。

出力先: `~/Desktop/ask-network-YYYY-MM-DD-TOPIC.md`

ファイルの構造:

```markdown
# [テーマ] に関する紹介・相談メッセージ集

---

## 0. 汎用投稿文（Facebook / Slack 用）

[汎用メッセージ]

---

## 1. [人物名]さん宛て

[パーソナライズドメッセージ]

---

## 2. [人物名]さん宛て

...

---

## 補足: 元のコンテキスト（参考用）

[元のメールや相談内容]
```

出力後、ユーザーに「`~/Desktop/ask-network-...md` に書き出しました」と報告する。

## 対応プロジェクト

| プロジェクト名 | 用途 |
|---|---|
| `tkgshn-private` | 個人の人脈（デフォルト） |
| `plural-reality` | チームの人脈 |

両方を検索し、より関連性の高い情報を使う。

## 注意事項

- **プライバシー**: Scrapbox の人物ページには個人的なメモが含まれる。メッセージに「Scrapbox で調べた」とは書かない。自然に知っている体で書く
- **サブエージェント活用**: 人物の背景調査は Agent ツールで並列実行する。5人なら5並列
- **ユーザー確認**: 候補者リストはユーザーに確認してからメッセージ作成に進む（ただしユーザーが既に名前を指定している場合はスキップ）
- **一人称**: ユーザーの普段の言い回しに合わせる（デフォルト: 「僕」）
- **ファイル出力必須**: ターミナルに直接出力しない。必ずファイルに書き出す
- **connect.sid 期限切れ**: エラーが出たらユーザーに通知

## このスキルがやらないこと

- メッセージの実際の送信（送信はユーザーが行う）
- 相手の連絡先の検索（Scrapbox に書いてあれば補足として提示する程度）
- 相手の返答の予測や戦略立案（壁打ちが必要なら `/kabeuchi-coach` を使う）
